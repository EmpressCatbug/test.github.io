Overview

This project demonstrates my ability to develop, organize, and test backend services using Java and JUnit as part of CS 320. Acting as a software engineer for Grand Strand Systems, I created modular service classes for a mobile application and applied comprehensive unit testing to ensure functionality and quality assurance.

Services Developed

I developed three independent services ‚Äî Contact, Task, and Appointment ‚Äî each with their own object model, service manager, and unit tests. These backend components handled creation, updating, and deletion without any UI layer.

  • Contact Service: ID, name, phone, address with validation
  • Task Service: Task name and description with character limits
  • Appointment Service: Date-time scheduling with conflict prevention

Unit Testing Strategy

I implemented a full suite of JUnit test cases to validate functionality and constraints. The tests included positive cases, exception handling, and edge validation (e.g., nulls, ID duplication). Coverage exceeded 80%, ensuring code reliability and regression safety.

Example Test Snippet


@Test

    ContactService service = new ContactService();
    Contact contact = new Contact("123", "Takeria", "5551234567", "Galaxy Blvd");

}

Software Design & Modularity

The services were structured using object-oriented principles. Each class was modular, independent, and easy to extend. I separated logic into reusable methods and grouped related functionality into their own services ‚Äî ensuring scalability and easier maintenance.
